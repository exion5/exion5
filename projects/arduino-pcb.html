S1 · Overview

The goal was to design a fully self-contained microcontroller board that could replace an Arduino Nano in the RC transmitter project since they must be wired to power on. My idea was to create one with integrated LiPo charging and battery management, removing the need for an external charger. Everything from USB connectivity to 5V boost conversion had to fit on a small 4-layer board.

The board went through multiple schematic review iterations before layout, every subsystem (power, USB, protection, charging) was verified independently before being connected together.

S2 · Power Architecture

The power system has four cascaded stages: USB/battery input → battery protection → LiPo charger → boost converter → 5V rail → LDO → 3.3V rail. Each stage was selected and calculated individually.

The TPS61022 boost converter generates 5V from the LiPo's 3.0–4.2V range. The output voltage is set by a resistor divider on the feedback pin. The TPS61022 uses a 0.6V internal reference, so the divider was calculated as:

Math TPS61022 feedback divider 
# TPS61022 feedback equation:
# V_out = V_ref × (1 + R10/R11)
# V_ref = 0.600V,  target V_out = 5.0V

# Rearranging:
# R10/R11 = (V_out / V_ref) - 1
#         = (5.0 / 0.6) - 1
#         = 7.333

# Choose R11 = 100 kΩ (standard value)
# R10 = 7.333 × 100 kΩ = 733.3 kΩ → use 732 kΩ (nearest E96)

R11 = 100 kΩ
R10 = 732 kΩ   # → V_out = 0.6 × (1 + 732/100) = 4.992V ✓

S3 · Battery Protection — DW01A + FS8205A

The DW01A is a LiPo protection IC that monitors cell voltage and current, cutting off the FS8205A dual MOSFET on overcurrent, overdischarge, or overcharge. The key wiring detail: the negative battery terminal (-BATT) must be kept as a separate net from GND through the protection circuit. Merging them prematurely bypasses the protection entirely.

Common mistake Connecting -BATT directly to GND on the schematic is a silent error — DRC won't catch it, the board will appear to work, but the overcurrent and overdischarge protection is completely bypassed. The FS8205A must sit in series with the negative battery path.

S4 · PCB Layout — Trace Widths & USB Routing

Trace widths were calculated using the IPC-2221 standard for a 1 oz copper, 10°C temperature rise target. The USB D+/D− differential pair was routed on the same copper layer with matched lengths to avoid signal skew — critical for reliable USB 2.0 full-speed communication.

Layout priority The TPS61022 boost converter and its inductor were placed first, co-located with the shortest possible SW node trace. High-frequency switching nodes radiate EMI proportional to loop area — minimising the distance between the IC switch pin, inductor, and output cap was the highest-priority placement constraint on the board.

S5 · Prototypes

Rev 1 was a minimal first pass, the power system used a simple LM1117MP-5.0 linear regulator fed from an external supply, with no battery management, no charging, and no protection circuitry. The focus was getting the ATmega328P, FT232RL USB bridge, and pin headers correct before tackling power complexity. Essentially, it was a proof-of-concept for the basic functionality and acted identically to a traditional Arduino Nano.

1 / 2

Rev 2 (current) replaced the entire power section with a proper battery-powered architecture: a TPS61022 boost converter for 5V output from a LiPo cell, MCP73831 for USB charging, and DW01A + FS8205A for overcurrent and overdischarge protection.
