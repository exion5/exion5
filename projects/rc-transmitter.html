S1 · Overview

The goal was a fully functional RC transmitter with no off-the-shelf radio module doing the heavy lifting — just an Arduino Nano, an NRF24L01+LNA transceiver, and hand-written firmware talking SPI. The transmitter handles two analog joystick axes, a rotary encoder for auxiliary control, an MPU-6050 IMU for tilt sensing, and an I²C LCD for live status display.

S2 · Power Design — Level Shifting

The Arduino Nano runs at 5V logic. The NRF24L01 and MPU-6050 both run at 3.3V and are not 5V tolerant on their data lines. Connecting them directly would damage the peripherals. The solution was a BSS138-based bidirectional logic level shifter — a MOSFET circuit that translates signals between voltage domains safely in both directions using only passive components.

The 3.3V rail is supplied by an AMS1117-3.3V LDO from the 5V Arduino supply. The AMS1117 can supply up to 800 mA, comfortably powering the RF module and IMU simultaneously.

Design note The NRF24L01+LNA variant has a built-in low-noise amplifier and PA stage, boosting range significantly over the standard module — but it draws more current (~115 mA TX peak), which is why the LDO selection mattered. The AMS1117's 800 mA headroom handles it without issue.

S3 · Firmware — Packet Structure

The transmitter firmware reads all inputs, packs them into a fixed-size struct, and fires it over SPI to the NRF24 every 20 ms (50 Hz update rate). Using a struct ensures the receiver can unpack the exact same memory layout without fragile byte-by-byte parsing.

C++ transmitter.ino 
// Radio packet — matched exactly on receiver side
struct RadioPacket {
  int16_t  throttle;    // joystick Y1  (-512 to 511)
  int16_t  yaw;         // joystick X1
  int16_t  pitch;       // joystick Y2
  int16_t  roll;        // joystick X2
  int16_t  tilt_x;      // MPU-6050 accel X
  int16_t  tilt_y;      // MPU-6050 accel Y
  uint8_t  aux;         // rotary encoder position
  uint8_t  flags;       // bit 0 = arm, bit 1 = mode
};

RadioPacket pkt;
RF24 radio(9, 10);   // CE, CSN pins

void loop() {
  pkt.throttle = analogRead(A0) - 512;
  pkt.yaw      = analogRead(A1) - 512;
  pkt.pitch    = analogRead(A2) - 512;
  pkt.roll     = analogRead(A3) - 512;

  // Read IMU over I2C
  readMPU6050(pkt.tilt_x, pkt.tilt_y);

  pkt.aux   = encoder.read();
  pkt.flags = (armSwitch ? 0x01 : 0x00);

  radio.write(&pkt, sizeof(pkt));
  delay(20);   // 50 Hz
}
